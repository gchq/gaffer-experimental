get /gaas-rest-service/graphs
(gaasRestServiceGraphsGet)
The GET /graphs endpoint returns an array of graphs which exist in the cluster. Each graph has basic information about it such as its name, description and store type.

delete /gaas-rest-service/graphs/graphName
(gaasRestServiceGraphsGraphNameDelete)
The DELETE /graphs endpoint allows users to delete graphs they have created. Simply enter the name of the graph you wish to delete and send the request. This will then delete the graph as well as all its associated resources in the cluster.

post /gaas-rest-service/graphs
(gaasRestServiceGraphsPost)
The POST /graphs endpoint allows users to create graphs. You must provide a graph ID, description, config name and schema in order to create successfully. This will then spin up a new graph instance in the cluster.

Consumes

This API call consumes the following media types via the Content-Type request header:
  • application/json

Request body

body gaasrestservice_graphs_body (optional)
Body Parameter —
example: { "value" : "{\n \"graphId\":\"mytestgraph\",\n \"description\":\"swagger\",\n \"configName\":\"mapStore\",\n \"schema\": {}\n}" }

Responses

get /gaas-rest-service/storetypes
(gaasRestServiceStoretypesGet)
The /storetypes endpoint returns a list of the storetypes supported by GaaS along with the paramters which are required by each storetype. If you are creating a graph directly using the API this is a good starting point to work out what paramters you will need to pass when creating a request from the /graphs endpoint.

Return type

inline_response_200_1

Example data

Content-Type: application/json
{
  "storeTypes" : [ {
    "name" : "name",
    "parameters" : [ "parameters", "parameters" ]
  }, {
    "name" : "name",
    "parameters" : [ "parameters", "parameters" ]
  } ]
}
